# 优化 issue_unit_list.vue 文件

## 问题分析

1. **代码结构问题**：
   - 文件过长（1400+行），包含大量模板和脚本代码
   - 逻辑混杂，没有清晰的模块化
   - 模板复杂，包含多个弹窗和表单

2. **代码质量问题**：
   - 变量命名不规范（如 `formeEitVisible` 拼写错误）
   - 存在冗余代码（如重复调用接口）
   - 缺少注释，特别是复杂业务逻辑
   - 函数过长（如 `GetUnitConfigsApi`）
   - 硬编码配置项索引
   - 日期处理逻辑复杂

3. **可维护性问题**：
   - 配置项管理混乱
   - 表单验证规则分散
   - 未使用组件拆分
   - 状态管理不够清晰

4. **功能问题**：
   - 密码重置功能未完整实现
   - 日期处理使用自定义函数

## 优化计划

### 1. 代码结构优化

- **组件拆分**：
  - 将主要弹窗拆分为独立组件（编辑弹窗、配置弹窗、二维码弹窗）
  - 将复杂表单拆分为子组件

- **模块化脚本**：
  - 将 API 调用封装为独立模块
  - 将业务逻辑按功能分类组织
  - 使用组合式函数（composables）管理相关状态和逻辑

- **模板优化**：
  - 简化主模板结构
  - 使用插槽和子组件减少重复代码

### 2. 代码质量优化

- **命名规范**：
  - 修正变量命名错误
  - 使用语义化变量名

- **代码清理**：
  - 移除冗余代码
  - 合并重复逻辑

- **注释完善**：
  - 添加文件头部注释说明功能
  - 为复杂函数添加详细注释
  - 为配置项添加说明注释

- **函数拆分**：
  - 拆分过长函数为多个子函数
  - 每个函数只负责单一功能

- **配置管理**：
  - 使用常量或枚举替代硬编码索引
  - 集中管理配置项

- **工具函数优化**：
  - 优化日期处理逻辑
  - 提取通用工具函数

### 3. 可维护性优化

- **状态管理**：
  - 使用更清晰的状态组织方式
  - 减少不必要的响应式状态

- **表单验证**：
  - 统一表单验证规则
  - 使用更简洁的验证方式

- **错误处理**：
  - 完善错误处理逻辑
  - 统一错误提示方式

### 4. 功能优化

- **完善密码重置功能**：
  - 实现完整的密码重置流程

- **日期处理**：
  - 考虑使用日期库（如 dayjs）简化日期处理

- **性能优化**：
  - 减少不必要的 API 调用
  - 优化数据处理逻辑

## 预期成果

- 文件结构清晰，模块划分合理
- 代码可读性高，注释完善
- 功能完整，无冗余代码
- 可维护性强，易于后续修改和扩展

## 实施步骤

1. 分析现有代码结构和功能
2. 设计优化后的代码结构
3. 拆分组件和模块化脚本
4. 优化代码质量和可维护性
5. 完善功能和性能优化
6. 测试确保功能正常
